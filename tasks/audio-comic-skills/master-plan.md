# Master Plan：有声漫画自动化生产 Skills 体系
## v0.6 | 2026-04-24

---

## 一、项目概述

### 核心场景
**来料加工**：输入是原著文本（小说、超长篇），输出是有声漫画产品。

### 目标
构建一套可被任意 CLI LLM 理解的 Skills 体系，触发词**"开始创作有声漫画"**后自动完成全流程。

## 三、Skills 协作关系

### 3.1 触发链路

```
用户输入："开始创作有声漫画"
              ↓
    Skill 3 编排引擎
              ↓
    Skill 1 查询知识库（获取参考）
              ↓
    Skill 2 风格锚定（保持一致）
              ↓
    7 环节流水线执行
         ↓         ↓         ↓
    脚本生成    分镜设计    生图
         ↓         ↓         ↓
    配音       合成       排版
              ↓
    Skill 4 监督验收（每个环节包裹）
              ↓
    Skill 5 良品率优化（批次结束后）
              ↓
    Skill 0 更新任务书
```

### 3.2 数据流

```
输入：原著文本
   ↓
Skill 1：知识库查询 → 返回参考资料
   ↓
Skill 2：风格锚定 → 返回风格参数
   ↓
Skill 3：流水线执行
   ├── 脚本 → 分镜 → 生图 → 配音 → 合成 → 排版 → 发布
   └── 每环节 Skill 4 监督验收
   ↓
Skill 5：良品率分析 → 输出优化建议
   ↓
Skill 0：任务书更新
   ↓
输出：有声漫画产品
```

## 四、六大 Skills 详细说明

### Skill 0: task-book-keeper
| 属性 | 内容 |
|------|------|
| **功能** | 任务书管理、核心记忆、每周审视 |
| **触发** | 任务开始/结束、每周审视 |
| **输入** | 任务更新内容 |
| **输出** | 任务书版本更新 |

### Skill 1: knowledge-base-manager
| 属性 | 内容 |
|------|------|
| **功能** | 知识库管理、增量式 Wiki |
| **触发** | 查询参考、添加知识 |
| **输入** | 查询内容 |
| **输出** | 结构化知识 |
| **数据源** | 得到笔记 API [网络] + GitHub [网络] |

### Skill 2: comic-style-consistency
| 属性 | 内容 |
|------|------|
| **功能** | 风格锚定、声音一致 |
| **触发** | 生成角色、保持画风 |
| **输入** | 风格要求 |
| **输出** | 风格参数 |
| **依赖** | reference image 池 |

### Skill 3: audio-comic-workflow
| 属性 | 内容 |
|------|------|
| **功能** | 7 环节流水线编排 |
| **触发** | "开始创作有声漫画" |
| **输入** | 原著文本 |
| **输出** | 各环节产物 |

### Skill 4: supervision-anti-drift
| 属性 | 内容 |
|------|------|
| **功能** | 监督验收、漂移检测 |
| **触发** | 自动包裹所有环节 |
| **输入** | OpenSpec 规范 |
| **输出** | 验收报告 |

### Skill 5: self-optimizing-yield
| 属性 | 内容 |
|------|------|
| **功能** | 良品率优化、经验沉淀 |
| **触发** | 批次结束后自动 |
| **输入** | 批次产物 |
| **输出** | 良品率报告、优化建议 |

## 五、模型选型规则

### 成本优先原则

| 任务类型 | 优先模型 | 备选模型 |
|----------|---------|---------|
| 文本生成 | DeepSeek V3 | Claude Sonnet |
| 图像生成 | 通义万相 | DALL-E |
| 语音合成 | 火山引擎 | ElevenLabs |
| 代码生成 | DeepSeek Coder | Claude |

### 单任务单模型原则

每个任务只使用一个模型，避免模型切换开销。

## 八、验收机制

### NCA 必要条件

| 环节 | 必要条件 | 阈值 |
|------|---------|------|
| 脚本生成 | 字数误差 | <10% |
| 分镜设计 | 场景完整 | 100% |
| 生图 | 风格一致 | ≥0.85 |
| 配音 | 情感准确 | ≥0.8 |

### Checkpoint 机制

每个环节输出 checkpoint，失败可回滚。
