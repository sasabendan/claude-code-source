二、训练 AI 的完整思路：从 PDF 到智能创作系统
目标：将静态知识库转化为会进化的智能系统，以《小李飞刀》漫画项目为例，构建包含
人物形象、剧情分解、文件管理、反馈优化的闭环体系。
(一) 知识库文件结构
核心架构：小李飞刀漫画项目文件夹下分设
- 漫画创作/：存放分回内容（如第一回/第二回）
- 人物形象/：中央大脑，含李寻欢.md、阿飞.md 等角色档案
- 工作流说明书.md：AI 必读的项目规则集
- 配角索引.md：角色出场与退场管理
(二) 六大训练步骤
1. 锁定人物形象
◦ 目标：AI 生成符合古龙风格的国风水墨武侠形象
◦ 方法：
① 提取小说人物特征
② 上传电影截图参考（狄龙版李寻欢、尔冬升版阿飞）
③ 反复迭代至风格统一
◦ 关键规则：人物形象单独建文件夹，AI 创作前必须“报到”调用
2. 剧情分解
◦ 内容：原文页码、章节概述、出场人物、场景描述（地点/对话/分镜建议）
◦ 作用：规定后续创作的格式标准与人物状态
3. 建立文件管理系统
◦ 核心规则：
▪ 新角色出场 → 先创建形象卡片
▪ 角色退场 → 更新索引
▪ 所有文件设置双链，实现人物-剧情-场景联动
4. 反馈优化
◦ 数据对比：
| 阶段 | 成品数 | 废片数 | 成本 | 风格达标率 | |——|——–|——–|——|————| | 第
一回 | 42 张 | 60+ 张 | 30 刀 | 50% | | 第四回 | 38 张 | 10+ 张 | <10 刀 | 70% |
◦ 方法：创建标准画风参考文件和作品风格指南，明确满意设计与禁止风格
5. 创建“中央大脑”
◦ 载体：工作流说明书.md，记录所有重要指示
◦ 规则：所有 AI 开始工作前必须阅读此文件，避免“幻觉”
6. 创建专属 skill
◦ 将项目要求整合为 feidao skill，触发命令“创作小李飞刀漫画”即可自动执行全
流程
三、复利系统：构建活的知识网络
核心思想：通过结构化本地 markdown 文件，让AI 清晰掌握项目权限、职能和进度。
实践方法：
- 物料、写作、书籍、思考分文件夹存放
- 每个内容独立为markdown 文件

- 重复流程 → 创建 skill
底层逻辑：与爆火的 Hermes agent 一致，实现知识库从“死wiki”到“活系统”的转

四、Git 版本控制：AI 创作的安全网
问题：AI 误删文件或覆盖满意版本（如仅需修改台词却重生成整张漫画）。
解决方案：通过 Git 实现版本回溯与备份。
(一) 安装与配置步骤
1. 安装插件：Obsidian 第三方插件市场搜索“Git”并启用
2. 初始化仓库：终端输入 git init ，设置分支 git branch -M main
3. 创建 GitHub 私人仓库：存储远程备份，确保数据隐私
(二) 连接本地与远程
通过 AI 执行配置命令：
添加远程仓库地址：git@github.com:用户名/仓库名.git
配置用户信息：Your Name <your@email.com>
提交初始化：git add . && git commit -m "Initial commit"
推送到远程：git push -u origin main
(三) 自动备份设置
• Auto commit-and-sync interval：设为 10 分钟
• 效果：实时备份+完整历史记录，支持任意版本回退
五、自动配图&排版：效率提升工具
需求：快速为文章添加风格统一的插图并排版。
方法：调用 baoyu skill 的 自动创作并插入配图 功能。

调用baoyu skill，整理文章为公众号版本，添加标题/分段，简洁配色，插入插图，在当前文件夹创建新文件
流程：选择风格→生成插图→自动排版，耗时约一首歌的时间。
扩展功能：支持一键发布至公众号和推特（需 API 调用，可能产生少量费用）。
