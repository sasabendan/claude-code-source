## 良品率追踪与复盘机制

### 定位
S5 self-optimizing-yield 的核心理念：批次结束后自动复盘，沉淀经验。

### 追踪维度
- 环节通过率（每步 NCA 是否达标）
- 漂移频率（哪个环节最容易漂）
- 产出良品率（图片合格数/总生成数）

### 复盘内容
- 本批次哪些环节出了问题
- 原因分析（drift / 风格不一致 / API 不稳定）
- 优化建议（改 prompt / 加 checkpoint / 换 API）

### 存储
`~/.claude/yield/records.jsonl`（JSONL，每批一条）

### 与 Supervisor 的关系
- Supervisor（S4）负责单环节验收
- Yield Optimizer（S5）负责批次级别的全局复盘
- 两者数据互通：S4 的 checkpoint → S5 的复盘输入

### 原文位置
`skills/self-optimizing-yield/scripts/yield-optimizer.sh`（已实现）
