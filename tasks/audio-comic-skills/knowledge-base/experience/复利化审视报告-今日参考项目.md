### 项目 1：LangExtract（SCRIPT-EXTRACTOR）

#### 复利化评分

| 维度 | 评分 | 依据 |
|------|------|------|
| 主线相关性 | ⭐⭐⭐⭐⭐ | Stage 0 剧本预提取，直接服务 P1 脚本生成 |
| 复利效应 | ⭐⭐⭐⭐ | char_interval 溯源 → 知识库持久积累 |
| 实现复杂度 | 🟡 中 | 需 API key，但脚本框架已就绪 |
| 替代风险 | 🟡 低 | 手工提取可替代，但质量/效率差 10x |

**复利层级：L1 基建层**（直接提升主任务能力）

#### 核心洞察

LangExtract 的价值不在于"提取"本身，而在于 **char_interval 溯源机制**：
- 每个 extraction 都有 `[start, end]` 位置 → 原文永远可查
- 4 类提取物（character/dialogue/scene/sfx）→ 结构化数据跨 Agent 共享
- grounding_rate ≥ 95% 阈值 → 知识库积累质量有保障

#### 与 kb-rust 对接的最优路径

```
小说文本（原始）
    ↓ langextract-pre.sh（预提取）
extractions.jsonl（4类结构化）
    ↓ grounding-verify.sh（溯源验证）
kb-rust ingest <extraction>（持久积累）
    ↓
kb-rust query（知识复用）
    ↓
P2/P3/P4 流水线调用
```

**当前缺口**： grounding-verify.sh 脚本存在，但 langextract-pre.sh 尚未完整实现（API key 待配置）。

#### 复利化判断

✅ **立即整合价值高**。char_interval 溯源是知识库积累的核心机制，LangExtract 的 4 类结构化模板（SCRIPT-EXTRACTOR.md）已写入规范文档。

#### 最优动作

- [ ] 完成 langextract-pre.sh（配置 API key，测试单章节）
- [ ] 将 extraction 结果与 kb-rust ingest 对接（测试角色/场景入 kb）
- [ ] 建立 grounding_rate 监控 → 触发 S5 优化回路

#### 核心洞察

qmd 对 Rust 知识库最有价值的 4 个工程思路：

| qmd 特性 | 借鉴点 | 在 kb-rust 中的位置 |
|---------|--------|------------------|
| BM25 + Vector + RRF | 混合检索精度提升 | 待实现 |
| Context Tree | 世界观层级上下文 | kb-rust 的 experience/ 目录可映射 |

| AST-aware Chunking | 语义完整性分片 | 已有 LangExtract 可替代 |

#### 关于"世界线收束"的本质澄清

**用户核心洞察**（来自对话）：
> "我不需要写世界线收束算法。逻辑判断由 LLM 承担，工程约束由任务系统承担。"

这意味着：
- **不需要**：复杂的图算法、因果推理引擎
- **需要**：可靠的 **grounding 溯源**（查知识库）+ **Rigor Gaps 检测**（逻辑自洽）= 伪"世界线收束"

qmd 的 Context Tree 正是为此服务：建立"路径即分类"的层级上下文，让 AI 在处理分镜时天然具备全局视野。

#### 与 kb-rust 的对接

```
qmd context add "qmd://characters" → kb-rust kb add <character> type=character
qmd context add "qmd://scenes" → kb-rust kb add <scene> type=scene  
qmd context add "qmd://plotlines" → kb-rust kb add <plotline> type=plotline
```

**关键**：kb-rust 不需要替代 qmd，需要"吸收其逻辑"。

#### 复利化判断

✅ **立即整合价值高**。BM25+RRF 混合检索是 kb-rust 当前最缺失的能力，世界观层级上下文建立直接服务 P2 分镜设计。

#### 最优动作

- [ ] 在 kb-rust v2 中增加 BM25 关键词搜索（不依赖向量数据库）
- [ ] 实现 RRF 融合（BM25 结果 + 标签匹配 + 时间衰减）
- [ ] 建立 experience/ 层级目录（characters/scenes/plotlines/timelines）
- [ ] C1 Rust 升级时考虑 SQLite-Vec（qmd 同款向量库）

## 六、核心结论

### 今日参考项目的复利化真相

经过审视，今日 5 个参考项目可分为 3 类：

**A 类：直接整合，立即复利（3个）**
- compound-engineering：ce-compound 飞轮 + Plan-Heavy → 已整合 ✅
- claude-mem：PostToolUse 蒸馏 + Progressive Disclosure → kb-rust 升级方向 ✅
- qmd：混合检索 + Context Tree → kb-rust 升级方向 ✅

**B 类：有价值但需等待（1个）**
- LangExtract：grounding 溯源核心，但 langextract-pre.sh 未完成 → API key 配置后立即执行

**C 类：参照借鉴，不直接整合（1个）**
- pi-mono：TypeScript 异构环境，提取设计思想用于 C1 Rust 升级

### 关于"世界线收束"的最终判断

**不需要写复杂算法**。真正的"世界线收束"通过以下机制实现：
1. **grounding 溯源**（LangExtract char_interval）→ 原文可查
2. **知识库层级**（qmd Context Tree 思路）→ 全局视野
3. **逻辑校验**（Rigor Gaps 4维检测）→ 自洽性确认

这三者结合 = 伪"世界线收束"，在工程上完全可落地。

### kb-rust 的工程定位再确认

| 用户期望 | kb-rust 实际能力 |
|---------|----------------|
| 知识库帮我提取小说文本 | ✅ kb-rust ingest + LangExtract 预提取 |
| CC 监督 + Codex 执行双保险 | ✅ 三记忆文件 + 确权四步 |
| 防上下文爆炸 | ✅ 三层索引（Progressive Disclosure）|
| 防逻辑跑偏 | ✅ Rigor Gaps 检测层 |
| 世界线收束 | ✅ grounding 溯源 + 知识库层级上下文 |

**kb-rust 的定位**：不是"搜索工具"，而是"知识编译系统"（LLM Wiki）。它让 AI 每次都在已有合成知识上工作，而非每次重新推导。
