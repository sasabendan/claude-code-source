## 核心论点

**Karpathy 方法**：静态知识库（收集、链接、检索）
**本文方法**：动态智能系统（理解、创造、进化）

关键金句：
> "从此，你的本地知识库可不仅仅是收集资源的地方，它是你的第二大脑，更是会持续进化的贾维斯。"

## 实践案例：小李飞刀漫画创作

- 过去1个月：Claude + Obsidian 完成 200+ 张漫画
- 起点：仅一份原著 PDF
- 成果：AI 对创作标准的完全理解

### 六大训练步骤

1. **锁定人物形象**：上传电影截图参考，反复迭代至风格统一，单独建文件夹，AI 创作前必须"报到"调用
2. **剧情分解**：原文页码、章节概述、出场人物、场景描述（地点/对话/分镜建议）
3. **建立文件管理系统**：新角色出场先创建形象卡片，角色退场更新索引，双链联动
4. **反馈优化**：第一回 42 张成品/60+废片/30刀/50%达标 → 第四回 38 张成品/10+废片/<10刀/70%达标
5. **创建"中央大脑"**：`工作流说明书.md`，AI 开始工作前必须阅读，避免幻觉
6. **创建专属 Skill**：触发命令即可自动执行全流程

### 知识库文件结构

```
小李飞刀项目/
├── 漫画创作/       # 分回内容（第一回/第二回）
├── 人物形象/       # 中央大脑，李寻欢.md、阿飞.md 等
├── 工作流说明书.md  # AI 必读的项目规则集
└── 配角索引.md     # 角色出场与退场管理
```

### 反馈数据

| 阶段 | 成品数 | 废片数 | 成本 | 达标率 |
|------|--------|--------|------|--------|
| 第一回 | 42张 | 60+张 | 30刀 | 50% |
| 第四回 | 38张 | 10+张 | <10刀 | 70% |

关键洞察：
- "AI批量处理文件的能力越强，你就越需要版本控制来兜底。"
- 每次修改 AI 都会更理解作者需求，AI 对话次数越多幻觉概率越高，Obsidian + 本地知识库可有效降低风险

## 版本控制

- 工具：Obsidian Git 插件 + GitHub 私人仓库
- 自动提交间隔：10 分钟
- 目的：AI 误删/覆盖满意版本时，可随时回退

## 自动排版配图

- 工具：baoyu-skills 的"自动创作并插入配图"功能
- 调用方式："调用baoyu skill，整理文章为公众号版本，添加标题/分段，简洁配色，插入插图"
- 支持一键发布公众号和推特

## 核心差异对比

| Karpathy 方法 | 本文方法 |
|--------------|----------|
| 静态知识库（收集、链接、检索） | 动态智能系统（理解、创造、进化） |
| 静态 Wiki | 活的系统，会持续进化 |
| 信息收集 | 训练 AI 理解用户风格、标准、审美 |
| 被动检索 | 主动陪伴进化 |

## 对本项目的设计影响

1. **中央大脑文件**：每个项目需要一个"工作流说明书.md"，AI 开工前必读
2. **人物形象管理**：角色单独建文件夹，创作前必须调用指定形象
3. **剧情分解标准化**：每章创建剧情分解文件，含页码、概述、出场人物、场景描述
4. **反馈优化循环**：记录满意/禁止风格，不断调整提升达标率
5. **专属 Skill**：将项目要求整合为触发命令，自动执行全流程
6. **Git 版本控制**：必须，为 AI 误操作留安全网
