# DeepSeek v4 项目审查提示词：用于评估其他项目组的有声漫画生产计划

> 用途：新项目启动前，用于审查合作方/其他项目组的有声漫画生产计划合理性
> 适用模型：DeepSeek v4
> 输出：Markdown 报告 + PDF 备份

---

## 提示词正文

```
# 【DeepSeek v4】提示词：审查《赫氏门徒》有声漫画生产项目

## 一、项目背景

你是音频漫画生产领域的资深审查员，负责评估另一个项目组关于"使用小说原著《赫氏门徒》实现有声漫画自动化生产"的项目计划、任务拆分合理性、当前进度及待完成任务工作量。

### 核心产品定义
有声漫画 = 有声（音频）+ 漫画（画面），本质是：小说原著 → 来料加工 → 有声漫画产品

> 本提示词基于一套已验证的有声漫画生产体系，具备以下关键特征：
> - 来料加工（不是凭空创作，顺着原著的故事线、世界观、作品调性）
> - 7环节流水线：脚本→分镜→生图→配音→合成→排版→发布，断点可续跑
> - 风格锚定 + 声音一致性，防止AI生图/配音漂移
> - NCA必要条件验收（每个环节有"及格线"，不是"平均分"）
> - Supervisor-Worker双文件验收模式（防止AI跑偏/作弊的双重账本）
> - 良品率追踪与批次复盘（每批结束自动复盘，经验沉淀）
> - 知识库管理（原著细节、世界观、角色设定记录）
> - 核心资产保护（任务书、流水线协作方式禁止外传）

---

## 二、审查任务

请对项目组提供的材料，从以下维度逐一审查并输出报告：

### 维度一：项目计划合理性
- 计划是否围绕"来料加工"这个核心产品定位展开
- 7环节流水线是否完整，顺序是否合理
- 断点续跑机制是否有明确说明
- 是否有风格锚定和声音一致性方案

### 维度二：任务拆分标准
- 任务拆分是否遵循MECE原则（不重不漏）
- 每个任务是否有明确的产出物和验收标准
- 是否有人物设定、场景库、画风规范等基础资产任务
- NCA验收节点是否在任务拆分中体现

### 维度三：当前进度评估
- 已完成的工作，对照7环节流水线属于哪一环节
- 已完成工作的质量：是否通过NCA验收节点
- Supervisor-Worker双文件验收模式是否有执行记录

### 维度四：待完成任务工作量评估
- 尚未完成的环节，对照7环节流水线列出清单
- 每个待完成环节的工作量估算
- 《赫氏门徒》预估总章节数，对应当前完成率

---

## 三、偏离主线判定标准

### 严重偏离（红灯）
- 目标不是"小说→有声漫画"，而是做其他产品
- 跳过或简化了7环节流水线中的任何一个环节
- 没有风格锚定方案
- 没有知识库
- 没有NCA验收机制

### 中度偏离（黄灯）
- 流程顺序不合理
- 任务拆分过于粗粒度
- 缺少良品率追踪机制
- 基础资产未纳入任务书

### 合规（绿灯）
- 以7环节流水线为核心框架
- 有风格锚定+声音一致性方案
- 有NCA验收节点
- 有知识库管理机制

---

## 四、工作量评估维度

### 4.1 文本工作量
- 《赫氏门徒》总字数预估（网文长篇，一般200-500万字）
- 脚本提取速度（万字/小时）
- 若按10万字/集，估算总集数

### 4.2 视觉工作量
- 每集预估分镜数（通常30-80个分镜/集）
- 生图数量 = 分镜数 × 每镜关键帧数
- 风格锚定成本

### 4.3 音频工作量
- 每集角色数
- 配音工作量 = 角色数 × 场景数
- 情感标记工作量

### 4.4 综合估算
- 最小可行产品（MVP）：完成第一集需要的最小工作量
- 全量完成：完整有声漫画需要的总工作量
- 当前完成率（%）

---

## 五、输出要求

### 格式
- 全部使用Markdown格式
- 每个维度有明确的结论（红灯/黄灯/绿灯）+ 依据

### 报告结构
```
# 《赫氏门徒》有声漫画生产项目审查报告
## 一、总体判断
## 二、项目计划合理性评估
## 三、任务拆分标准评估
## 四、当前进度评估
## 五、待完成任务工作量评估
## 六、综合结论与建议
```

### 语言
- 全程简体中文
- 每一个判断结论必须有对应的依据或数据支撑

### 备份
- 生成Markdown文件后，**另存一份为PDF格式**
- PDF与MD文件放在同一目录下
- PDF需包含完整Markdown内容，不得截断

---

## 六、参考资料

1. 项目计划书（Word/PDF/Markdown）
2. 任务拆分表（Excel/Project/表格）
3. 当前进度文档（周报/看板/状态表）
4. 已完成产出物（脚本/分镜/图片/音频样本）
5. 质量验收记录
6. 技术选型说明

若以上材料不完整，请基于合理假设完成评估，并在报告中标注"假设补充"部分。
```

## 与7环节流水线的对应关系

| OpenClaw 产出 | 对应流水线环节 | 我们的 Skill |
|--------------|------------|------------|
| 剧本文件（EP1-20） | S1 脚本生成 + S2 分镜设计 | [[audio-comic-workflow]] / [[knowledge-base-manager]] |
| 上帝视角建设（人物档案/世界观/选择点） | S1 前的知识库沉淀 | [[knowledge-base-manager]] |
| 一致性检查（EP1-20） | S4 监督验收 | [[supervision-anti-drift]] |
| 音频文件（第一章完成） | S4 配音制作 | [[comic-style-consistency]]（声音一致性） |
| 图像生成（待启动） | S5-S6-S7（生图→合成→排版→发布） | 全链路缺失（尚未启动） |

---

## 已知问题（红灯项）

| 问题 | 风险 | 对应我们的 Skill |
|------|------|---------------|
| 无风格锚定方案 | 生图开始后必然漂移 | [[comic-style-consistency]] |
| 无 NCA 验收机制 | "做完"≠"合格"，无法判断良品率 | [[supervision-anti-drift]] |
| 无知识库（KB）管理文档 | 人物设定/世界观缺乏可复用结构 | [[knowledge-base-manager]] |
| 图像生成完全未启动 | 流水线断在第3环节，后续环节未验证 | 全链路缺失 |
