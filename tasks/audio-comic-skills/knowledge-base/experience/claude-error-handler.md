## FC004 根因分析

**事件**：对 TASK_REQUIREMENTS.md / TASK_PROGRESS.md / master-plan.md 执行加密推送后，误删本地明文文件。

**错误链条**：
1. 误解 C11：「备份必须加密」被理解为「加密备份完成后删除本地」
2. C11 原文：`本地备份不加密，做好版本管理` —— 本地永远是明文
3. GitHub 加密备份 ≠ 删除本地：两者是并行关系，不是替代关系
4. 未触发 C17：删除前未查任务书/知识库，直接执行高危操作
5. 未触发 C19：删除后无自检记录

**正确流程**：
```
① 加密文件并推送 GitHub  →  ② 本地 .md 明文保留不动  ← 这一步被跳过
```
而不是：
```
① 加密推送  →  ② 删除本地 .md  ← 错误步骤
```

**教训**：
- 任何对「本地文件生命周期」的修改，都必须先查 C17（C11 原文在任务书中）
- 知识库管理的文件删除 = 高危操作，必须 C17 查询 + 红灯禁止自动删除
- 「加密备份完成」≠「本地文件使命结束」
