## 一、项目概述

### 核心场景
**来料加工**：输入是原著文本（小说、超长篇），输出是有声漫画产品。

### 生产流程思维
```
【来料】原著小说文本
    ↓
【Step 1】顺着原著的故事线、世界线、作品调性 → 打磨剧本
    ↓
【Step 2】细节标注 + 知识库搭建 → 尊重原著
    ↓
【Step 3】流水线分工协作 → 分镜/生图/配音/合成/排版
    ↓
【Step 4】产出合格产品 → 有声漫画
    ↓
【持续】良品率追踪 → 不断提升 → 质量可靠的作品
```

### 关键词
| 关键词 | 含义 |
|--------|------|
| **来料加工** | 基于原著二次加工，不是凭空创作 |
| **尊重原著** | 顺着故事线、世界观、作品调性 |
| **流水线** | 分工协作，不是 AGI 协商 |
| **良品率** | 从合格开始，不断提升 |
| **核心资产** | 流水线协作方式 + 核心技能优化 + **任务书**，禁止外传 |
| **任务书迭代** | 每周审视，持续优化 |

## 二、交付物功能总览

| 组合功能 | 说明 |
|---------|------|
| **自身任务管理** | 任务书管理、进度追踪、记忆保留与迭代 |
| **全流程自动化** | 脚本→分镜→生图→配音→合成→排版→发布，全自动可中断续跑 |
| **风格一致性** | 画面风格锚定 + 声音音色固定，跨生成保持一致 |
| **多模型智能调度** | 单任务单模型，成本优先 |
| **监督验收机制** | checkpoint 回滚，NCA 必要条件阻断 |
| **自我优化能力** | 良品率追踪，经验沉淀，防退化保护 |

## 五、每周任务（Weekly Review）

### 触发条件
**每周一次**，在每个自然周结束时执行。

### 任务内容

```
## 每周审视任务

### 1. 审视主线目标
- 前一周对主线目标（来料加工+流水线）的理解是否有更新？
- 是否有新的认知需要补充到任务书中？

### 2. 审视 Skill 架构
- 当前 7 个 Skills 的分工是否合理？
- 是否有功能重叠或缺失？
- 是否有新的参考来源需要添加？

### 3. 审视执行进度
- 各 Skill 开发的实际进展 vs 计划
- 遇到的问题及解决方案
- 风险评估

### 4. 主动优化
- 识别任务书中的不合理之处
- 提出优化方案
- 更新任务书版本

### 5. 核心记忆更新
- 记录本周的关键理解
- 更新核心记忆文档
- 与任务书同步
```

## 八、执行约束

1. **逐 Skill 参考逐个搭建**：每个 Skill 必须先精读对应参考来源，再产出 SKILL.md
2. **已有 Skills 优先复用**：先集成当前仓库已有 Skills，再开发新能力
3. **禁止臆造 API**：所有外部接口必须基于参考资料核实
4. **参考资料访问失败**：明确报告并提出替代方案
5. **版本锁定不可协商**：任何自动更新脚本必须排除 openspec
6. **备份优先**：任何工具调用前先确认备份机制有效
7. **核心资产保护**：任务书 + 流水线协作方式 + 核心技能优化参数禁止外传
8. **每周审视**：每周末执行任务书审视（C13）
