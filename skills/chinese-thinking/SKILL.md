# Skill: chinese-thinking（中文语言思维锚定）

## 定位

强制中文思维锚点。所有任务执行前必须先用中文完成"三段式总结"，确保中文作为主思维语言。

## 触发时机

本 Skill **无条件触发**，每次收到用户消息后，在执行任何操作前必须先输出三段式总结。

## 三段式总结格式（硬性要求）

收到用户第一条消息后，**立即输出**：

```
【中文思维锚定】
用户要我做什么：<1-2句话描述>
我计划用什么步骤完成：<步骤列表>
我还缺什么信息：<缺失信息或"暂无"]
```

完成这段输出后，才能开始执行具体任务。

## 挂钩范围

| 场景 | 触发条件 |
|------|---------|
| 技能管理/优化 | 涉及 Skills 创建/修改/测试 |
| 知识库管理 | 涉及 KB 操作（add/search/ingest）|
| 主线任务 | 涉及 heartbeat-state / 主线进度 |
| 复利化决策 | 涉及参考项目评估/整合 |
| C 系列约束 | 涉及 C17-C23 查询顺序 |

## 硬性要求

1. **三段式是第一个输出**：必须在任何工具调用、任何代码生成之前
2. **使用中文标点**：句号用"。"，逗号用"，"，顿号用"、"（中文全角）
3. **字数控制**：三段式总计 50-150 字，简洁为要
4. **不省略任何一段**：即使"暂无"也要写

## 正确示例

```
用户：帮我优化 kb-rust 的 search 命令，增加 BM25 检索

【中文思维锚定】
用户要我做什么：优化 kb-rust 的 search 命令，增加 BM25 混合检索功能。
我计划用什么步骤完成：①阅读现有 search 命令源码 ②设计 BM25 积分逻辑 ③修改 main.rs ④测试验证 ⑤更新 SPEC.md
我还缺什么信息：当前 search 命令的具体实现逻辑（需读取 main.rs）
```

## 错误示例

```
用户：帮我优化 kb-rust 的 search 命令

❌ 直接执行（跳过三段式）
   → 违反 C17 查询顺序

❌ 用英文输出三段式
   → 违反语言锚定要求

❌ 三段式过于简略（<20字）
   → 缺乏思考深度

❌ 省略"我还缺什么"
   → 违反硬性要求
```

## 实现

`scripts/chinese-anchor.sh`：三段式总结生成脚本，供 SessionStart Hook 调用或手动触发。

## 与 C17 查询顺序的关联

三段式总结中的"我还缺什么"直接服务于 C17 查询顺序：

```
三段式"我还缺什么" → 对应 C17 第①步（任务书）
                              ↓
                          第②步（知识库）
                              ↓
                          第③步（上下文）
                              ↓
                          第④步（技能）
                              ↓
                          第⑤步（本地备份）
                              ↓
                          第⑥步（网络搜索）
                              ↓
                          第⑦步（带经验问用户）
```
