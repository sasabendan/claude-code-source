# Skill: chinese-thinking（中文思维锚定）

## 定位

**思考启动锚点，不是生产检验环节。**

每个任务执行前，必须先确认"我要交付什么"，把注意力聚焦在产出物上，而不是过程理解上。

## 触发时机

本 Skill **无条件触发**，每次收到用户消息后，在执行任何操作前必须先输出三段式。

## 三段式格式（硬性要求）

收到用户第一条消息后，**立即输出**：

```
【主线锚定】
任务要求产出：<具体产物描述（含文件路径/数量/格式）>
当前产出状态：【有/无/部分】
与目标的差距：<具体不足，或"已达标">
```

完成后才能开始执行具体任务。

## 三段式填写说明

| 字段 | 填写规范 |
|------|---------|
| **任务要求产出** | 必须具体：含文件名、数量、格式。不写"优化""改进"等模糊词。 |
| **当前产出状态** | 三选一：【有】【无】【部分】。不是主观判断，是客观对比。 |
| **与目标的差距** | 若【有】→写"已达标"；若【无】或【部分】→写具体差距是什么。 |

## 硬性要求

1. **三段式是第一个输出**：在任何工具调用、代码生成之前完成
2. **使用中文标点**：句号用"。"，逗号用"，"，顿号用"、"
3. **"当前产出状态"三选一**：禁止写"不知道""不清楚"——先查任务书再判断
4. **"与目标的差距"必须具体**：若【部分】→至少写一条不足

## 正确示例

```
用户：帮我把 chinese-thinking 的索引同步功能写成独立脚本

【主线锚定】
任务要求产出：skills/chinese-thinking/scripts/sync-index.sh（可执行脚本）+ 更新 .index.jsonl
当前产出状态：【无】
与目标的差距：脚本不存在；.index.jsonl 条目待更新（chinese-thinking updated 字段 17:00 < 17:18）
```

```
用户：检查知识库索引状态

【主线锚定】
任务要求产出：.index.jsonl 全部条目的 updated 字段 ≥ 对应 SKILL.md 修改时间
当前产出状态：【部分】
与目标的差距：chinese-thinking 索引 updated=17:00 < file=17:18（待同步）
```

## 错误示例

```
❌ 任务要求产出：优化脚本
   → 模糊，应写具体脚本名和路径

❌ 当前产出状态：应该差不多有
   → 非三选一，应为【有/无/部分】

❌ 与目标的差距：缺少一些细节
   → 不具体

❌ 与目标的差距：暂无（状态为【无】时）
   → 状态为【无】必有差距，不允许写"暂无"
```

## 定位说明

| 本 Skill | 不是 |
|---------|------|
| 思考启动锚点 ✅ | 生产线检验环节 ❌ |
| 确认"交付什么" | 定义"怎么做" |
| 产出物导向 | 过程理解导向 |

**本 Skill 不绑定知识库查询、不强制 C17 步骤、不替代 skill-change-test.sh。**

检验工作由 skill-change-test.sh 承担（机器验证，客观可靠）。本 Skill 只负责让每次执行从"交付什么"开始。

## 实现

`scripts/chinese-anchor.sh`：三段式生成脚本，供手动触发。
